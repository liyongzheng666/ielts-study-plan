## 任务A：生成学习记录

1. 读 `学习模版/ielts-study-outcome-polar-bears.md`，照它的章节结构产出**7 类成果**：
   ① 精读同义替换批注(5–8组) ② 单词卡 6–8 张(Anki格式) ③ 同义替换组入总库
   ④ 句型库 2–3 个 ⑤ 30 秒复述 ⑥ 配套题+逐题归因 ⑦ 写作Task2或口语范例+批注 ⑧ 总结。
2. 先忠实转录原文到「〇、阅读原文」，再据此生成成果。
3. 文件名按规范：`C{册}-T{套}-P{篇}-{文章中文名}[-partN].md`，放进 `学习记录/第N个月/`。
   - 能从材料判断册/套/篇就带上；判断不了先问我一句出处，别瞎编。
   - 同篇拆多次精读用 `-part1/-part2`，并加文件间双向交叉链接。

## 任务B：同步推送（必做，除非我明说"先别推"）

1. 维护 `assets/app.js` 的 `studyRecords`（对应月份 `sessions` 增改 `{session,label,path}`）。
2. 更新 `README.md` 的「学习记录」链接。
3. 自检：`node --check assets/app.js` 通过；`grep` 无旧文件名残留引用。
4. `git mv` 保留历史；单次提交（信息讲清做了什么+为什么），`git push origin main`。
5. 等 GitHub Pages 构建完成后用 `curl` 验证新文件 URL→200、旧 URL→404；必要时 grep 线上 app.js 确认 label 已更新。

最后用简短中文汇报：生成了哪个文件、网页验证结果（状态码）、线上地址。
